# Dukungan SaringSini

SaringSini adalah proyek open-source tahap awal yang dipelihara secara best effort. Panduan ini membantu memilih kanal yang tepat tanpa menjanjikan waktu respons atau penyelesaian tertentu.

## Pertanyaan penggunaan

Gunakan [template pertanyaan](https://github.com/SultanZhalifa/SaringSini/issues/new?template=question.yml) untuk pertanyaan setup, perilaku aplikasi, atau cara berkontribusi. Cari issue yang sudah ada terlebih dahulu dan sertakan konteks secukupnya.

Jangan menyertakan API key, isi percakapan pribadi, file sensitif, atau data pribadi dalam issue publik.

## Laporan bug

Gunakan [template bug report](https://github.com/SultanZhalifa/SaringSini/issues/new?template=bug_report.yml). Sertakan langkah reproduksi, perilaku yang diharapkan dan aktual, serta lingkungan yang digunakan. Bersihkan log dan screenshot dari rahasia atau data pribadi sebelum mengunggahnya.

## Proposal fitur

Gunakan [template feature request](https://github.com/SultanZhalifa/SaringSini/issues/new?template=feature_request.yml). Jelaskan masalah yang ingin diselesaikan dan periksa [roadmap](ROADMAP.md) serta issue terbuka agar proposal tidak menduplikasi pekerjaan.

## Laporan keamanan

Jangan membuat issue publik untuk kerentanan. Ikuti kanal privat yang tercantum di [SECURITY.md](SECURITY.md). Private vulnerability reporting GitHub belum diaktifkan untuk repositori ini, sehingga jangan mengandalkan formulir Security Advisory sebagai kanal aktif.

## Ekspektasi respons

Maintainer akan meninjau pertanyaan, bug, proposal, dan laporan keamanan sesuai kapasitas dan prioritas. Tidak ada SLA untuk konfirmasi, triage, perbaikan, atau rilis. Laporan keamanan dan risiko data pengguna akan diprioritaskan secara best effort.

## Batas dukungan

- Keluaran AI bukan verifikasi resmi dan maintainer tidak dapat menjamin kebenaran setiap hasil.
- Proyek tidak menyediakan nasihat profesional untuk kesehatan, hukum, keuangan, atau keselamatan.
- Deployment produksi, database eksternal, kebijakan organisasi, dan pengelolaan cloud berada di luar dukungan dasar proyek kecuali ada dokumentasi khusus yang menyatakannya.
